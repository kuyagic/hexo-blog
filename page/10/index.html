使用 bash 脚本上传文件到 Google Drive

2012-09-13

之前介绍了使用 WebDAV 来进行备份.那毕竟支持WebDAV的服务并不多,虽然有,但是也有各种各样的限制,比如单文件大小之类的.万一备份文件比较大,那也是比较麻烦的存在.
所以,还是用Google吧.
Google Drive,前身GoogleDocs.
为了和Dropbox竞争,也推出了桌面客户端.但是如何从Linux Shell脚本来进行上传呢?

那就要万能的Python童鞋出场了.

使用这个Python脚本可以把指定文件传到Google Drive下的指定文件夹下.使用方法如下







gupload [filename] [collection]

这里的 collection 就是Google Drive的根目录下的文件夹名字,如果不指定 就是默认传到根目录
那我们这么使用一下







gupload /www/backup.tar.gz “vpsbackup”

那这个就是把 /www/backup.tar.gz 文件上传到Google Drive的 vpsbackup 的文件夹下了.

自动化备份服务器数据到WebDAV

2012-09-11

由于现在VPS价格越来越平民化,越来越多的人有了VPS,那有独立服务器是好.但是备份数据也是自己的责任了.

那这里我就来说说怎么用cronjob来定时备份需要的数据. 备份目标可以是ftp,amazon s3 等.

最近一直在研究WebDAV,那这个文章就用webdev来做说明.

首先由于VPS大多都采用的是Linux系统,这里以Debian 6来做说明.

先安装webDAV客户端







apt-get install cadaver

然后建立一个自动登陆的配置文件







vi ~/.netrc

内容格式如下







machine [webdavserver]
login [username]
passwd [password]

上面的例子里
webdavserver就是备份目标的服务器地址,可以从这里参考免费的服务
下面两行分别是用户名和密码
然后保存这个文件
再在shell里执行







cadaver [webdavserver]

就可以自动登陆到dav服务器了.

最近在 Mac 和 Windows 下尝试了 Sublime Text 2 之后，发现它非常强大好用，更新也非常快。推荐大家使用。它是共享软件，不购买也可以完整使用，只是有时候保存的时候会提示你购买，取消就可以了。

Sublime Text 2 的特色功能：

良好的扩展功能，官方称之为安装包(Package)。
这条来说 我装了以下扩展
安装扩展的办法,先安装Sublime Package Control
方法在编辑器里按







Ctrl + `

然后再输入以下语句直接回车,然后重启软件







import urllib2,os; pf=‘Package Control.sublime-package’; ipp=sublime.installed_packages_path(); os.makedirs(ipp) if not os.path.exists(ipp) else None; urllib2.install_opener(urllib2.build_opener(urllib2.ProxyHandler())); open(os.path.join(ipp,pf),‘wb’).write(urllib2.urlopen(‘http://sublime.wbond.net/‘+pf.replace(‘ ‘,‘%20’)).read()); print ‘Please restart Sublime Text to finish installation’

就能在Preferences 里看到Package Control的菜单了
